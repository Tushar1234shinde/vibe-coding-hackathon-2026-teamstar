# Diagrams

This repository includes architecture and schema documentation for the platform.

- `docs/architecture.md` describes the high-level service layout.
- `sql/schema.sql` holds the relational ER schema.
- `neo4j/schema.cql` includes graph constraints and entity definitions.

## Suggested diagrams

- System architecture diagram: frontend, backend, DB, graph, vector store, cache, message queue, AI providers.
- ER diagram: user, problem, solution, research, submission, collaboration entities.
- Knowledge graph diagram: problems, subproblems, solutions, contributors, research relationships.
